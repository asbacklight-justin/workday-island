# 实时聊天与窗口互动

[中文](REALTIME_CHAT.zh-CN.md) · [English](REALTIME_CHAT.md)

## 使用方式

1. 点击主窗口顶部的聊天按钮。
2. 可修改显示昵称，然后点击“一键上线”。应用不会要求用户名或密码。
3. 首次上线后复制自己的用户 ID，并通过可信渠道发给对方。
4. 输入对方的用户 ID，即可发送文字消息，或发送“抖动窗口”“闪烁窗口”指令。窗口互动可以附带最多 120 个字符的提示语。
5. 点击“下线”会关闭实时连接，但保留本机匿名身份；“重置身份”会清除本机身份和聊天记录，下次上线将获得新的用户 ID。

对方暂时离线时，文字消息和窗口互动由服务端记录，并在对方下一次上线后补发。新消息到达时，聊天页会自动定位到最近发信人的会话。接收窗口互动时，工位岛会恢复、置前，展示发送者的提示语，并执行短暂抖动或彩色闪烁；用户可立即停止效果。

## 一键身份与安全

首次点击上线时，应用在本机生成 Ed25519 密钥对，并向 Backlight 实时服务注册匿名设备身份：

- 服务器只接收公钥、设备 ID、设备名称和显示昵称；
- 私钥不离开本机；
- macOS 私钥保存在系统钥匙串；
- Windows 私钥通过当前用户的 DPAPI 加密；
- 后续上线通过签名服务端随机挑战完成，不保存或传输用户名密码。

本机业务数据（待办、提醒、薪资、工作时间、天气设置和专注记录）不会发送到实时服务。

## 网络与故障排查

桌面端通过 `wss://admin.asbacklight.cn/api/realtime/ws` 建立加密 WebSocket 连接，并通过 `https://admin.asbacklight.cn/api/realtime/bootstrap` 完成首次匿名设备注册。

如果界面提示“实时服务网关暂未启用 WebSocket”，说明 HTTPS 网关没有正确转发 WebSocket Upgrade，而不是一键身份或密码错误。Nginx 代理至少需要：

```nginx
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
```

应用会在非认证故障后按 1、2、4、8、16、30 秒的节奏自动重连；主动下线后不会重连。
